# Estudo de Viabilidade - TCC

Este documento apresenta o estudo de viabilidade do projeto **SET CLI** (*Software Estimation Tool*), desenvolvido por Inácio Moraes da Silva como Trabalho de Conclusão de Curso do Bacharelado em Engenharia de Software da PUC Minas. O projeto consiste em uma ferramenta de linha de comando que utiliza Inteligência Artificial e análise de dados históricos do GitHub para auxiliar equipes ágeis na estimativa de esforço de tarefas de desenvolvimento de software.

## 1. Enquadramento na Resolução de TCC

Conforme estabelecido no Anexo I da Resolução do Trabalho de Conclusão de Curso, item 2, o trabalho final de TCC II pode consistir de pesquisa em Engenharia de Software ou desenvolvimento (implementação) de software. Este projeto se enquadra inequivocamente na segunda modalidade, focando no desenvolvimento de uma ferramenta CLI completa e funcional. Mais especificamente, o projeto atende ao previsto no Projeto Pedagógico do Curso (PPC) para trabalhos de desenvolvimento sem cliente específico, que devem consistir de ferramenta para engenharia de software alinhada aos parâmetros de Salão de Ferramentas ou Periódicos com ênfase em publicação de software.

O SET CLI atende a essa exigência ao apoiar diretamente o processo de desenvolvimento de software, auxiliando uma atividade crítica em metodologias ágeis: a estimativa de esforço de tarefas. A proposta se alinha com tópicos de interesse de eventos como o SBES (Simpósio Brasileiro de Engenharia de Software), particularmente seu Salão de Ferramentas, que abrange áreas como Engenharia de Software, Processos Ágeis, Inteligência Artificial Aplicada e Ferramentas de Apoio ao Desenvolvimento. Ao automatizar e tornar mais precisas as estimativas através do uso de IA e dados históricos, a ferramenta contribui concretamente para ganhos de produtividade no planejamento de sprints, alocação de recursos e cumprimento de prazos.

A demanda atendida pelo projeto é a dificuldade persistente de equipes ágeis em realizar estimativas precisas de esforço para tarefas de desenvolvimento de software. Técnicas tradicionais como Planning Poker apresentam subjetividade excessiva, dependendo fortemente da experiência individual dos membros da equipe e resultando em variabilidade significativa nas estimativas. Adicionalmente, equipes raramente mantêm bases de conhecimento organizadas sobre tarefas anteriores e seus esforços reais, desperdiçando aprendizado valioso. Não existem soluções consolidadas que combinem dados históricos do GitHub com análise por IA de forma acessível via linha de comando. Estimativas humanas também são suscetíveis a vieses cognitivos como ancoragem e otimismo excessivo. O SET CLI endereça essas lacunas ao combinar dados históricos reais do GitHub, algoritmos de similaridade textual para encontrar tarefas análogas e modelos de linguagem de grande escala (LLMs) para gerar estimativas baseadas em evidências com níveis de confiança quantificados.

## 2. Viabilidade Técnica e Risco Tecnológico

A viabilidade técnica do projeto está fundamentada na integração de componentes tecnológicos maduros e amplamente validados, demonstrando baixo risco tecnológico conforme esperado para um trabalho de conclusão de curso em desenvolvimento de software. A integração com a GitHub API (REST e GraphQL) utiliza interfaces estáveis e documentadas, com autenticação segura via Personal Access Tokens e gestão eficiente de rate limiting através de cache local e sincronização incremental. A persistência local é realizada através do BoltDB, um banco de dados key-value embarcado que não requer dependências externas, garantindo portabilidade total e oferecendo transações ACID com performance adequada para os volumes de dados esperados.

O motor de similaridade textual emprega algoritmos estabelecidos de Processamento de Linguagem Natural, especificamente TF-IDF (Term Frequency-Inverse Document Frequency), similaridade de cosseno e similaridade de Jaccard. Estes algoritmos são amplamente validados em sistemas de produção como motores de busca e sistemas de recomendação. A implementação de busca adaptativa com thresholds dinâmicos (variando de 50% a 15%) garante resultados mesmo para tarefas sem correspondências exatas nos dados históricos. A integração com APIs de Inteligência Artificial utiliza serviços comerciais maduros como OpenAI GPT-4 e Claude da Anthropic, ambos com documentação completa e ampla adoção. Importante ressaltar que o sistema opera corretamente mesmo sem configuração de API de IA, utilizando apenas dados históricos e média ponderada por similaridade como fallback.

A escolha da linguagem Go 1.21+ e arquitetura CLI oferece vantagens técnicas significativas para o projeto. Binários compilados nativamente proporcionam execução direta sem necessidade de runtime ou interpretador, enquanto goroutines e channels nativos permitem processamento paralelo eficiente através de worker pools para batch processing. A cross-compilation nativa para Linux, macOS e Windows garante portabilidade total sem modificações no código. Bibliotecas maduras e amplamente adotadas pela comunidade Go, como Cobra para CLI framework, Viper para configuração e testify para testes, reduzem significativamente o risco de implementação. Todos os componentes tecnológicos necessários são maduros, bem documentados e amplamente validados em ambientes de produção, confirmando que o risco tecnológico do projeto é baixo e centrado na engenharia de integração ao invés de pesquisa fundamental ou invenção de novas tecnologias.

## 3. Complexidade Técnica e Conhecimentos Mobilizados

Conforme estabelecido no Anexo II da Resolução de TCC, item 5, o trabalho deve ser executado de modo a obter resultados com solidez metodológica e qualidade técnica do desenvolvimento. Este projeto mobiliza um conjunto abrangente e aprofundado de conhecimentos, cobrindo múltiplos domínios da Engenharia de Software e demonstrando complexidade técnica adequada para um TCC de alto nível.

Na área de Inteligência Artificial Aplicada e Processamento de Linguagem Natural, o projeto envolve a implementação de algoritmos de similaridade textual (TF-IDF, similaridade de cosseno e Jaccard) para comparação semântica de tarefas, integração com modelos de linguagem de grande escala (LLMs) para análise contextual, aplicação de técnicas de prompt engineering para extração de estimativas com raciocínio explícito, e desenvolvimento de sistema de confiança quantitativa baseado em volume e qualidade de dados históricos. Em Arquitetura de Software, aplica-se Clean Architecture com separação clara entre camadas de domínio, aplicação, infraestrutura e apresentação, além de múltiplos padrões de projeto incluindo Strategy para múltiplos provedores de IA, Repository para abstração de persistência, Facade para simplificação de APIs complexas, e Chain of Responsibility para busca adaptativa.

No contexto de Engenharia de Software para Sistemas Distribuídos, o projeto envolve integração com APIs REST e GraphQL para comunicação com serviços externos (GitHub, OpenAI, Claude), implementação de tratamento robusto de rate limiting através de exponential backoff e circuit breakers, estratégias de cache e sincronização incremental para minimizar tráfego de rede, e processamento paralelo eficiente com worker pools implementados via goroutines. A persistência e gerenciamento de dados requer modelagem para key-value store com estruturação adequada de dados históricos em BoltDB, estratégias de serialização JSON para configuração e dados estruturados, e sistema robusto de configuração multi-fonte integrando arquivos, variáveis de ambiente e flags de linha de comando.

O projeto demonstra forte foco em qualidade de software através de cobertura de testes superior a 80% incluindo testes unitários, de integração e end-to-end, simulação de APIs externas através de mocks para testes rápidos e determinísticos, e documentação completa de 24 casos de teste (11 de aceitação e 13 de integração) rastreáveis aos requisitos. Aspectos de segurança são contemplados através de criptografia de tokens e API keys em configuração local, validação rigorosa de entrada para prevenção de injection attacks, e tratamento ético do uso de IA com transparência e explicitação de estimativas como sugestões ao invés de garantias.

Este conjunto de conhecimentos se relaciona diretamente com disciplinas fundamentais do curso: Arquitetura de Software através da aplicação integral de Clean Architecture e padrões SOLID; Desenvolvimento de Aplicações Distribuídas na integração com APIs RESTful e GraphQL; Inteligência Artificial no uso de algoritmos de NLP e integração com LLMs; Projeto de Software na aplicação de múltiplos padrões de projeto; Teste de Software nas estratégias de teste automatizado e cobertura de código; Gerência de Configuração e Evolução no Git workflow e versionamento semântico; e Processos de Software na compreensão profunda de metodologias ágeis e práticas de estimativa.

## 4. Conformidade com Critérios de Avaliação

O Anexo I da Resolução de TCC estabelece no item 6 que a banca avaliadora deve considerar cinco critérios fundamentais na avaliação do trabalho. O projeto SET CLI foi desenvolvido para atender integralmente a cada um destes critérios.

Quanto à lógica, sistemática e ordenação com começo, meio e fim, o projeto apresenta documentação estruturada em oito seções principais que conduzem o leitor desde a introdução e contextualização do problema, passando pela apresentação da solução e sua arquitetura, até os resultados dos testes e análise post-mortem. A fundamentação teórica, dados, exemplos e práticas são evidenciados através da base conceitual sobre estimativas ágeis e seus desafios, apresentação detalhada de algoritmos de NLP e suas aplicações, princípios de arquitetura de software aplicados no projeto, e validação através de casos de teste reais documentados.

A pertinência ao tema, atendendo à promessa do projeto inicial, é demonstrada pela implementação completa de todos os requisitos propostos no TCC I, mantendo fidelidade ao escopo original de criar uma ferramenta CLI para estimativas de esforço assistidas por IA. O projeto entrega uma solução que efetivamente resolve o problema objeto de pesquisa através de um sistema funcional que gera estimativas baseadas em dados históricos e IA, operacional e utilizável por equipes reais. A validação e avaliação da solução é comprovada através da execução completa de 24 casos de teste (11 de aceitação e 13 de integração) documentados na seção 6 da documentação, alcançando cobertura de código superior a 80%, e entrega de release v1.0.0 funcional demonstrando a viabilidade prática da solução proposta.

## 5. Relevância, Contribuição e Viabilidade de Execução

O projeto entrega valor concreto tanto para o meio acadêmico quanto para a indústria de software. Na dimensão acadêmica, constitui uma demonstração prática de aplicação de Inteligência Artificial em processos de engenharia de software, área de crescente interesse na pesquisa. Proporciona validação empírica de técnicas de similaridade textual especificamente para o contexto de estimativa de esforço, contribuindo para o corpo de conhecimento sobre aplicações de NLP em engenharia de software. O registro detalhado do processo de desenvolvimento incremental com foco em requisitos fornece material de referência para futuros trabalhos.

Para a indústria, o projeto oferece uma ferramenta prática e imediatamente utilizável por equipes reais de desenvolvimento, com potencial de adoção como código aberto para uso e evolução pela comunidade. A implementação em Go seguindo boas práticas estabelece um benchmark de qualidade técnica. A combinação única de integração com GitHub, análise de similaridade textual e uso de IA via interface de linha de comando representa inovação na forma de entrega, atendendo especificamente desenvolvedores que preferem ferramentas de terminal.

Quanto à viabilidade de execução, o projeto foi desenvolvido seguindo cronograma incremental de seis etapas alinhado aos marcos acadêmicos estabelecidos na Resolução de TCC: Infraestrutura e Configuração em setembro de 2025, Sincronização GitHub entre setembro e outubro, Motor de Estimativas em outubro, Processamento em Lote entre outubro e novembro, Relatórios e Análise em novembro, e Testes e Qualidade em novembro. O sistema encontra-se funcional com release v1.0.0, incluindo todos os casos de uso principais implementados e testados, demonstrando concretamente a viabilidade de execução dentro do prazo acadêmico estabelecido.

## 6. Conclusão

O projeto SET CLI demonstra total conformidade com a Resolução do Trabalho de Conclusão de Curso da PUC Minas. Enquadra-se adequadamente como ferramenta para engenharia de software que atende a uma demanda real identificada pelo aluno, mobiliza conhecimentos profundos de múltiplas áreas da Engenharia de Software (Inteligência Artificial, Arquitetura de Software, APIs, Testes, Sistemas Distribuídos), e apresenta complexidade técnica apropriada para um TCC de alto nível acadêmico.

A viabilidade técnica foi comprovada através da implementação bem-sucedida utilizando tecnologias maduras e bem documentadas, incluindo linguagem Go, GitHub API, OpenAI API, BoltDB e algoritmos estabelecidos de Processamento de Linguagem Natural. O risco tecnológico manteve-se baixo ao longo do desenvolvimento, centrado na engenharia de integração ao invés de pesquisa fundamental ou invenção de novas tecnologias. O projeto atende integralmente aos cinco critérios de avaliação estabelecidos pela banca examinadora: lógica e sistemática, fundamentação teórica, pertinência ao tema, solução funcional do problema, e validação através de testes.

O trabalho entrega valor concreto tanto para o meio acadêmico, através da demonstração prática de aplicação de IA em processos de engenharia de software, quanto para a indústria, fornecendo uma ferramenta funcional e utilizável por equipes reais. A execução seguiu cronograma incremental alinhado aos marcos acadêmicos, resultando em sistema funcional (v1.0.0) que comprova a viabilidade prática da proposta. Conclui-se que o projeto cumpre integralmente os objetivos de um Trabalho de Conclusão de Curso de Engenharia de Software, demonstrando capacidade técnica, rigor metodológico e relevância tanto acadêmica quanto profissional.
